# Security Policy

## Supported Versions

Security updates are currently applied to the latest version of DocSnap.

| Version | Supported |
|--------|----------|
| Latest | Yes      |
| Older  | No       |

---

## Reporting a Vulnerability

If you identify a security issue, report it responsibly.

### Do Not
- Open public GitHub issues for security vulnerabilities  
- Share exploit details publicly before resolution  

### Do
- Email: abduays2@gmail.com  
- Include a clear description of the issue  
- Provide steps to reproduce  
- Attach screenshots or proof of concept if applicable  

---

## What to Expect

- Acknowledgement within 48 hours  
- Initial assessment of severity and impact  
- Updates on progress where applicable  
- Notification once the issue is resolved  

---

## Scope

This policy applies to:

- Chrome extension source code  
- Data handling within the extension  
- External scripts or integrations  
- Permissions and access control  

Out of scope:

- Issues caused by outdated browsers or unsupported environments  
- User-side misconfigurations  

---

## Security Principles

Contributions and development must follow:

- Least privilege access  
- No unnecessary permissions  
- No external scripts without strict validation  
- Minimal data collection and storage  
- Clear handling of user-generated content  

---

## Data Handling

DocSnap is designed to minimise risk:

- No sensitive data should be stored unnecessarily  
- Avoid transmitting user data externally  
- Any storage must be local and controlled  
- Logs must not expose private information  

---

## Dependency Management

- Keep dependencies minimal  
- Regularly review and update dependencies  
- Avoid unverified or unmaintained packages  

---

## Responsible Disclosure

- Report issues privately  
- Allow time for fixes before disclosure  
- Do not exploit vulnerabilities beyond proof of concept  

---

## Summary

Security is enforced through minimal permissions, controlled data handling, and strict validation of all external interactions. All contributions must maintain or improve this standard.
